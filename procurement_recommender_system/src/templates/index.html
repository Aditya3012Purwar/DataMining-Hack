How the Recommendation Engine Works
Two-stage recommender system

With 2.7M SKUs and 18M transactions, this uses a candidate generation + ranking pipeline — the same architecture as Amazon/Netflix.

Stage 1 — Candidate Generation

The buyer's E-class purchase history filters the catalogue to same-category, not-yet-purchased items. Shrinks millions to a manageable pool.

Stage 2 — Ranking

final_score = P(buy) x (1 + price_advantage)
P(buy) combines: global popularity + manufacturer familiarity + price advantage

What "Saved %" means

Saved % = (ref_price - candidate_price) / ref_price
The reference price is this buyer's own average paid price for the same E-class — not a global average.

Column meanings
  • Cand. EUR — average market price of the alternative item
  • Ref. EUR — what this buyer currently pays on average for this E-class
  • Saved % — actual price difference vs buyer's own reference
  • P(buy) — estimated purchase probability (0-1)
  • Score — combined rank: higher = more likely AND cheaper
